Project 1: Powerise Production Analysis Tool

July 9, 2021

Objective: Reduce the number of reworks required in the Powerise production process by creating a visualization tool to summarize the capability of a production build and help stakeholders root cause build/machine issues.

Outcome: The visualization tool successfully summarizes the capability of builds in a visual and easy to understand way, and is now a required G5 deliverable for all Stabilus programs. The tool has been used to identify production build issues and has led to significant changes to the production control plan, such as:

  • Changing the Extended Length re-zeroing from once per shift to once per Workorder. This has led to reducing the number of reworks needed from 5% to <1% for some parts.
  • Requiring brakes to have pre-production run-in, making their behavior more consistent. This has led to reducing the number of reworks needed from 6% to 2% for some parts.

Process:

  1. Import Data
    Created a data pipeline that imported manufacturing data directly from the manufacturing center’s SQL database into Python.
  2. Clean Data
    Cleaned, filtered, and grouped the data to make the data easier to work with.
  3. Feature Engineering
    Using the imported data, created a class to automatically assign drawing values, and separate by End Of Line tester used.
  4. Output
    Graphed key parameter’s distribution, median, and drawing limits along side listing key production KPIs (Cpk, Cp), based on each workorder and end of line tester.

Examples:
Problem 1: Workorder values are inconsistent and End Of Line Testers are inconsistent - machine is inconsistent.

Resolution 1: Process change to make the machine more consistent. Increase re-zeroing of End Of Line Tester in control plan from once per shift to once per Workorder.

Problem 2: Workorder values are inconsistent but End Of Line Testers are aligned - parts are inconsistent.

Resolution 2: Process change to make the parts more consistent. Require pre-production brake run-in.
